# ドキュメント更新ガイドライン

このガイドラインは、ドキュメントの更新方法を定義します。あなたはこれに従ってドキュメントを更新してください。

## ドキュメントの構造

このプロジェクトでは、ドキュメントファイルを機能ごとに分割することで、実装に必要な情報だけを簡単に把握できることを目標としています。
これを実現するために、以下のような構造になっています。

```
├── docs
│   ├── README.md (どのような機能のフォルダがあるのか、各機能をそれぞれ1文で説明した索引)
│   ├── architecture.md (全体の構成)
│   └── 各機能のフォルダ
│       └── 必要があればサブフォルダを作成
│           ├── README.md (機能の概要、目的、注意点など)
│           ├── requirements.md (機能の要件)
│           └── implementations.md (実装の概要や関連するコードのファイルパス等を記載する。基本的にコード本体は記載しないが、特に注意すべき実装を数行引用して解説することは可能)
├── CLAUDE.md (AI向けの概要、このプロジェクトの目的やドキュメントの場所、開発を円滑に進めるためのコーディング規約や便利なコマンドのガイドなど。実装ステップや完了後のエラーチェック、ドキュメント更新手順を記載)
└── README.md (人間向けの概要、セットアップ手順や開発に参加する方法など)
```

## 注意点

1. 全てのファイルは200行以内に収めて、もしこれをオーバーするようなら適切に分割してください
2. 変更履歴や実装計画は記載しないでください
3. 未来の開発者が分かりやすいように、構造化された文章を心がけてください
